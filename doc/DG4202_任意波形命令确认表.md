# DG4202 任意波形命令确认表

此表用于 v0.4 前置确认。任意波形上传必须先确认 SCPI 命令，再写 driver。

## 当前状态

已确认：

- `*IDN?` 可通过 PyVISA/NI-VISA 访问 DG4202。
- 普通信号控制已在 WaveBench 中实现：function、frequency、Vpp、output、square duty。
- DG4000 系列属于函数 / 任意波形发生器，硬件类别支持任意波形。
- 公开产品规格显示 DG4202 / DG4000 任意波长度为 16K，垂直分辨率 14-bit，采样率 500MSa/s；这只能作为 payload guard，不等于上传 SCPI 已确认。

未确认：

- 任意波形上传命令。
- 上传数据格式。
- 用户波形存储 / 选择命令。
- 播放采样率或频率语义。

## 命令候选，不可直接当作已确认命令

以下只是其他任意波形发生器或 RIGOL 系列中常见的命名方向，不能直接写入正式 driver：

```text
DATA:DAC
SOURce<n>:DATA
SOURce<n>:FUNC:USER
SOURce<n>:FUNC ARB
SOURce<n>:APPLy:USER
VOLATILE
```

## query-only 探针

WaveBench 已有只读探针：

```bash
python -m wavebench source arb-probe --config wavebench.toml --channel 2
```

它只发送以 `?` 结尾的候选查询，并在每条查询后读取 `SYST:ERR?`。不上传数据，不选择波形，不打开输出。

当前候选：

```text
:SOUR2:FUNC?
:SOUR2:FUNC:USER?
:SOUR2:FUNC:ARB?
:SOUR2:ARB?
:SOUR2:ARB:SRAT?
:SOUR2:ARB:FREQ?
:SOUR2:DATA:CAT?
:SOUR2:DATA?
:DATA:CAT?
```

## 安全 probe 规则

- probe 前先读取并保存原状态。
- 默认不要打开输出。
- 如果必须输出，使用低风险参数：1 kHz、1 Vpp、0 V offset。
- 输出端接示波器，不接未知电路。
- probe 后恢复原状态。
- 每个候选命令后立刻读取 `SYST:ERR?`。

## WaveBench 当前离线中间格式

`wavebench source arb-load ... --dry-run --export-payload <path>` 会生成：

```text
format = wavebench.arbitrary.v1
encoding = dac14_unsigned_integer
range = 0..16383
values = full DAC point list
```

这只是 WaveBench 内部/人工复核用 payload，不等同于 DG4202 原生命令格式。正式 upload driver 仍需等 SCPI 确认。
