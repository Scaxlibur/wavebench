# DG4202 任意波形命令确认表

此表用于 WaveBench v0.4 前置确认。任意波形上传必须先经过离线 payload / SCPI block 生成、实机安全验证，再进入正式 driver。

## 当前接线

2026-05-01 起，实验台接线改为直觉一致：

```text
DG4202 CH1 -> RTM2032 CH1
DG4202 CH2 -> RTM2032 CH2
```

历史记录中若出现 `DG4202 CH2 -> RTM2032 CH1`，那是旧接线，不再作为当前默认路径。

## 当前状态

已确认（来源：DG4000 Programming Guide）：

- `*IDN?` 可通过 PyVISA / NI-VISA 访问 DG4202。
- 普通信号控制已在 WaveBench 中实现：function、frequency、Vpp、output、square duty。
- DG4000 系列属于函数 / 任意波形发生器，硬件类别支持任意波形。
- DG4202 / DG4000 任意波长度为 16K，垂直分辨率 14-bit，采样率 500 MSa/s；手册确认与 upload SCPI 一致。
- 任意波形上传 SCPI 命令已从编程手册确认。
- 上传数据格式已确认：DAC 14-bit binary / decimal，或 float voltage。
- 用户波形 U 盘存储/选择命令已确认。
- 播放频率语义已确认：无独立 `ARB:SRAT` 采样率命令，频率通过 `SOUR:FREQ` 或 `APPL:USER <freq>,...` 设置。

未确认（需要实机验证）：

- `DATA:DAC VOLATILE,#...` binary block 上传在 DG4202 实机上的行为。
- DAC 二进制块字节序已通过 DG4202 实机 `DATA:VAL?` 回读确认：little-endian 正确，big-endian 会按错位字节解释。
- `DATA VOLATILE,<val>,...` float voltage 格式在实机上的精度与限制。
- `FUNC:SHAP USER` 在上传 volatile arb 后的实际行为。
- 大波表 `DATA:DAC16` 多包上传。

## 已确认 SCPI 命令

以下命令来自 DG4000 Programming Guide，以 `TRACe`（波形上传/编辑）和 `SOURce`（选择/输出）两个子系统为核心。

### TRACe 子系统：任意波形上传 / 编辑

| 功能 | 完整命令 | 缩写 | 手册确认 | 备注 |
|---|---|---|---|---|
| 上传 14-bit DAC 二进制/十进制 | `[:TRACe]:DATA:DAC VOLATILE,[<binary_block>|<value>,<value>,...]` | `DATA:DAC VOLATILE,...` | 已确认 | 推荐 MVP 入口。DAC 范围 `0000`~`3FFF`，2~16384 点。二进制以 `#` 开头，十进制用逗号分隔。上传后自动切换当前通道输出易失波形。 |
| 上传 16-bit DAC 二进制（DDRII） | `[:TRACe]:DATA:DAC16 VOLATILE,<flag>,<binary_block>` | `DATA:DAC16 VOLATILE,<CON|END>,#...` | 已确认 | 固定 16 kpts / 32 kBytes。`CON`=还有后续包，`END`=最后一包。`END` 时自动切换到任意波输出。 |
| 上传 float 电压值 | `[:TRACe]:DATA[:DATA] VOLATILE,<value>{,<value>}` | `DATA VOLATILE,<val>,...` | 已确认 | 电压范围 -1~+1，单次 1~16384 点。上传后自动切换当前通道输出易失波形。 |
| 设置插值方式 | `[:TRACe]:DATA:POINts:INTerpolate LINear|OFF` | `DATA:POIN:INT LIN|OFF` | 已确认 | LIN=线性插值，OFF=阶梯保持。仅当前输出为易失波形时有效。 |
| 设置初始化点数 | `[:TRACe]:DATA:POINts VOLATILE,<value>|MIN|MAX` | `DATA:POIN VOLATILE,<val>` | 已确认 | 2~16384。自动切换当前输出为易失波形并初始化为 0。 |
| 修改单点 DAC 值 | `[:TRACe]:DATA:VALue VOLATILE,<point>,<data>` | `DATA:VAL VOLATILE,<pt>,<data>` | 已确认 | `<point>`=1~16384，`<data>`=0~16383。 |
| 查询单点值 | `[:TRACe]:DATA:VALue? VOLATILE,<point>` | `DATA:VAL? VOLATILE,<pt>` | 已确认 | 返回十进制整数值。 |
| 查询点数 | `[:TRACe]:DATA:POINts? VOLATILE[,MIN|MAX]` | `DATA:POIN? VOLATILE` | 已确认 | 返回 2~16384。 |
| 查询易失数据包数 | `[:TRACe]:DATA:LOAD? VOLATILE` | `DATA:LOAD? VOLATILE` | 已确认 | 返回数据包个数。 |
| 读取数据包 | `[:TRACe]:DATA:LOAD? <num>` | `DATA:LOAD? <num>` | 已确认 | 先查包数，再读第 num 个包。 |

### SOURce 子系统：任意波选择 / 输出

| 功能 | 完整命令 | 缩写 | 手册确认 | 备注 |
|---|---|---|---|---|
| 选择任意波输出 | `[:SOURce<n>]:FUNCtion[:SHAPe] USER` | `SOUR<n>:FUNC:SHAP USER` | 已确认 | `USER` 是 `FUNC:SHAPe` 参数，不是 `FUNC:USER?` 子树。 |
| 逐点输出模式 | `[:SOURce<n>]:FUNCtion:ARB:STEP` | `SOUR<n>:FUNC:ARB:STEP` | 已确认 | SET 命令，无参数，不是 query。 |
| 输出任意波（含参数） | `[:SOURce<n>]:APPLy:USER [<freq>[,<amp>[,<offset>[,<phase>]]]]` | `SOUR<n>:APPL:USER <freq>,<amp>,<offset>,<phase>` | 已确认 | 频率范围 1 μHz~40 MHz（实际受机型限制）。设置后切换任意波输出。 |
| 设置频率 | `[:SOURce<n>]:FREQuency[:FIXed] <frequency>|MIN|MAX` | `SOUR<n>:FREQ <freq>` | 已确认 | 任意波回放频率语义。采样率约等于点数 × 重复频率，且受 500 MSa/s 上限约束。 |
| 设置幅度 | `[:SOURce<n>]:VOLTage[:LEVel][:IMMediate][:AMPLitude] <amp>|MIN|MAX` | `SOUR<n>:VOLT <amp>` | 已确认 | 已有普通波形支持。 |
| 设置偏移 | `[:SOURce<n>]:VOLTage[:LEVel][:IMMediate]:OFFSet <offset>|MIN|MAX` | `SOUR<n>:VOLT:OFFS <offset>` | 已确认 | 已有 status 查询。 |
| 打开输出 | `:OUTPut[<n>][:STATe] ON|OFF` | `OUTP<n> ON` | 已确认 | 已有普通波形支持。 |

## 早期 probe 结果回译

2026-05-01 早期 query-only probe 的失败结论需要改写：失败原因是候选命令路径错误，而非 DG4202 不支持任意波。

| probe 命令 | 结果 | 手册对照 | 结论 |
|---|---|---|---|
| `:SOUR2:FUNC?` | `SIN` | `FUNC?` 返回当前波形名 | 普通 query 正常。历史 probe 用的是旧接线里的 CH2。 |
| `:SOUR2:FUNC:USER?` | `-113, Undefined header` | `USER` 是 `FUNC:SHAPe` 参数，无 `FUNC:USER?` | 命令不存在。 |
| `:SOUR2:FUNC:ARB?` | timeout / `0` | `FUNC:ARB:STEP` 是 SET 命令，无 query | 命令不存在或被静默忽略。 |
| `:SOUR2:ARB:SRAT?` | `-113, Undefined header` | 无独立采样率命令 | 使用 `SOUR:FREQ` / `APPL:USER`。 |
| `:SOUR2:DATA?` | `-113, Undefined header` | `DATA` 属于 `TRACe` 子系统，不属于 `SOURce` | 应使用 `:DATA:DAC VOLATILE,...`。 |

当前新接线下，后续验证应优先用 CH1：`SOUR1` / `OUTP1` / RTM2032 CH1。

## MVP 推荐上传 + 输出序列（当前 CH1）

安全起步时不要直接打开输出；先上传、查错、查询点数/单点。

```text
*CLS
DATA:DAC VOLATILE,#516384<binary_data>
SYST:ERR?
DATA:POIN? VOLATILE
DATA:VAL? VOLATILE,1
```

确认上传无错误后，再进入输出闭环：

```text
*CLS
DATA:DAC VOLATILE,#516384<binary_data>
SYST:ERR?
SOUR1:FREQ 1000
SOUR1:VOLT 1
SOUR1:VOLT:OFFS 0
SOUR1:FUNC:SHAP USER
OUTP1 ON
SYST:ERR?
```

或用 `APPL:USER` 合并设置：

```text
*CLS
DATA:DAC VOLATILE,#516384<binary_data>
SYST:ERR?
SOUR1:APPL:USER 1000,1,0,0
OUTP1 ON
SYST:ERR?
```

## WaveBench 当前离线中间格式

`wavebench source arb-load ... --dry-run --export-payload <path>` 会生成：

```text
format = wavebench.arbitrary.v1
encoding = dac14_unsigned_integer
range = 0..16383
values = full DAC point list
```

`--export-dg4000-dac-block <path>` 会额外写出离线 SCPI bytes：

```text
:DATA:DAC VOLATILE,#<digits><length><payload>
```

字节序状态：

```text
--dg4000-byte-order big|little
默认 little
状态：dg4202_hardware_validated_2026-05-01
```

注意：DG4000 编程手册确认每点两个字节和 DAC 码范围，但未明确二进制块字节序。2026-05-01 实机验证结果：DG4202 对 `DATA:DAC VOLATILE,#...` 使用 little-endian；big-endian 写入 `1024` 会回读为 `4`，little-endian 写入 `1024/2048/4096/8192/12288/16383` 均可按原值回读。

## 安全 probe 规则

- 默认不打开输出。
- 默认不改变用户当前波形设置。
- 任意会上传 payload 的命令必须有显式开关。
- 每步后读取 `SYST:ERR?`。
- 当前接线优先测试 CH1→CH1。
- 今天暂不测试 DP800 电源。

## 最小成功定义

v0.4 任意波形上传 MVP 只有在下面闭环成立后才算成功：

```text
上传 waveform → DG4202 CH1 输出 → RTM2032 CH1 捕获 → report / FFT 看到预期主频与波形形状
```
