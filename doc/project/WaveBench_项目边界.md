# WaveBench 项目边界

## 项目定位

WaveBench 是一个面向电赛调试场景的 Python 仪器自动测量工具，目标是在示波器、信号发生器和电脑之间建立轻量、可复现的测量流程。

第一阶段聚焦 R&S RTM2032 示波器的远程波形采集与数据保存。后续再扩展到普源信号发生器控制、简单测量流程编排和自动化实验记录。

一句话版本：

> WaveBench：给电赛调试用的轻量 Python 自动测量台。

## 要解决的问题

在电赛调试时，减少电脑、示波器和信号发生器之间的重复操作，让常见测量流程可以被脚本执行、记录和复现。

核心需求：

- 自动连接仪器；
- 远程执行示波器 Auto / Autoscale；
- 触发一次采集；
- 读取示波器波形；
- 保存波形数据；
- 保存测量元信息；
- 后续可自动设置信号发生器；
- 后续可运行固定测量流程。

## 阶段 1：示波器只读采集 + 显式 Autoscale

阶段目标：

> 可靠地让 RTM2032 自动调出稳定波形，读取当前波形，并保存数据和元信息。

### 必做

- 连接 RTM2032；
- 查询 `*IDN?`；
- 清状态 / 查询错误；
- 默认不重置仪器；
- 默认不主动改前面板主要设置；
- 提供显式 `auto / autoscale` 命令，对应手册 `AUToscale`；
- `auto / autoscale` 必须等待异步命令完成；
- 读取当前 CH1 波形；
- 读取波形 header；
- 构造时间轴；
- 保存 `.csv`；
- 保存 `.npy`；
- 保存 `.json` 元数据；
- 保存 `commands.log`；
- 提供最小 CLI：

```bash
wavebench scope idn
wavebench scope auto
wavebench scope fetch --channel 1
wavebench scope capture --channel 1
```

### 可做，但不强求

- 保存截图；
- 支持 CH2；
- 支持多通道逐个采集；
- 自动保存更完整的命令日志。

### 暂不做

- GUI；
- 实时波形显示；
- 自动配置复杂触发；
- 自动扫频；
- 信号发生器控制；
- 多仪器同步；
- 数据分析平台；
- 插件系统；
- Web 服务；
- 数据库。

## 阶段 2：示波器基础控制

在只读采集稳定后，加入有限控制能力：

- 设置通道开关；
- 设置垂直 scale / offset / coupling；
- 设置 timebase；
- 设置边沿触发；
- 单次采集 / 连续运行 / 停止；
- 支持 YAML 配置文件。

这一阶段才允许脚本接管仪器设置。

默认仍然保守：

```yaml
instrument:
  reset_before_run: false
  apply_front_panel_settings: false
```

## 阶段 3：信号发生器接入

普源信号发生器接入后，WaveBench 从“示波器采集脚本”扩展为“实验台工具”。

目标能力：

- 连接信号发生器；
- 设置正弦波 / 方波 / 三角波；
- 设置频率、幅值、偏置；
- 输出开关；
- 一键生成测试信号；
- 示波器采集响应波形；
- 保存输入配置和输出波形。

示例命令形态：

```bash
wavebench gen sine --freq 1k --amp 1.0
wavebench scope capture --channel 1
```

## 阶段 4：自动测量流程

在示波器和信号发生器都稳定后，再考虑实验流程编排。

可能支持：

- 扫频；
- 测增益；
- 测截止频率；
- 测峰峰值；
- 测频率；
- 测响应曲线；
- 自动生成 CSV / 图片 / 简单报告。

这一步很有用，但不能先做。先把 autoscale、fetch、capture 打牢。

## 明确不做什么

### 不做完整 SCPI 覆盖

WaveBench 不追求封装 RTM2032 的所有命令，只封装电赛和常见调试需要的命令。

### 不做通用仪器驱动库

WaveBench 第一目标是当前设备：

- R&S RTM2032；
- 普源信号发生器；
- 后续有需要再扩展别的设备。

它不是“支持所有示波器、所有信号源、所有协议”的通用库。

### 不替代前面板

示波器前面板调参很快。WaveBench 第一阶段不是替代旋钮，而是把“Auto、采集、保存、记录”这些高频动作自动化。

### 不做实时上位机

不追求毫秒级实时显示。数据采集是批处理式：

```text
Auto / 配置 → 采集 → 保存 → 分析
```

### 不把隐私和本机路径写进公开配置

默认配置使用 example。真实 IP、路径、实验数据和临时草稿放本地，不进 git。
